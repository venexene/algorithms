# Remove Duplicates from Sorted Array

## Варианты решения

### Two pointers (read/write) — O(n), O(1)
`write` указывает куда класть следующий уникальный элемент. `read` сканирует массив. Если `nums[read] != nums[read-1]` — новый уникальный: пишем в `nums[write]`, сдвигаем `write`. После цикла `write` равен количеству уникальных элементов.

## Что важно запомнить
- In-place: нельзя создавать новый массив. Перезаписываем элементы в том же слайсе.
- Всё что после `k` — мусор, LeetCode его не проверяет. Не нужно затирать.
- Указатель записи начинается с 1, а не с 0 — нулевой элемент всегда уникален.
- Никаких свапов, никакого поиска следующего бóльшего — только перезапись.
- Паттерн read/write встречается часто: Remove Element, Move Zeroes, Merge Sorted Array — все решаются похожим образом.

## Что пошло не так (итерации)
1. Искал следующее бóльшее значение вперёд — оставлял дыры, дубликаты возвращались
2. Считал замены вместо уникальных элементов — счётчик отставал на 1
