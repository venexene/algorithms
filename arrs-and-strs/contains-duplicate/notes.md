# Contains Duplicate

## Варианты решения

### 1. Хеш-таблица — O(n), O(n)
Проходим по массиву. Если число уже в мапе — дубликат найден. Если нет — кладём. Ранний возврат при первом повторе. В Go для сета идиоматично `map[int]struct{}` — `struct{}` занимает 0 байт, в отличие от `interface{}` или `bool`.

### 2. Сортировка + соседи — O(n log n), O(1)
Сортируем массив. Если есть дубликаты — они окажутся рядом.

## Что важно запомнить
- Компромисс: хеш-таблица быстрее, но ест память. Сортировка медленнее, но без доп. структур.
- `sort.Ints` — стандартная библиотека, не пиши сортировку руками если она не основная тема задачи
- Если массив маленький — сортировка может обогнать мапу из-за отсутствия аллокаций
- LeetCode замеряет время неточно — разница в пару мс между O(n) и O(n log n) на малых n — это шум
