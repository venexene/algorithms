# Longest Palindromic Substring

## Варианты решения

### Expand Around Center — O(n²), O(1)
Для каждой позиции раскрываем палиндром в обе стороны. Два вызова: нечётный (из `i, i`) и чётный (из `i, i+1`). Запоминаем максимальный по длине.

### Manacher — O(n), O(n)
Линейный алгоритм через preprocess строки (`#a#b#a#`) и переиспользование радиусов.

## Что важно запомнить
- Палиндромы бывают нечётные (`"aba"`) и чётные (`"abba"`). Нужны оба вызова expand.
- `expand` возвращает индексы ПОСЛЕ последней успешной проверки. Возвращай `(l+1, r-1)`, не `(l-1, r-1)` и не `(l, r)`.
- Цикл `i < len(runes)`, без `-1` — но чётный вызов защищён проверкой `i+1 < len(runes)`.
- Длина палиндрома: `r - l + 1`.

## Что пошло не так
1. Сначала учёл только нечётные — пропускал `"bb"`.
2. `expand` возвращал `(l-1, r-1)` вместо `(l+1, r-1)` — индексы убегали.
3. Цикл `i < len-1` ломал одиночные символы.
4. Чётный вызов `expand(i, i+1)` при `i = len-1` выходил за границу.

## Про Манакера
Алгоритм Манакера (1975) решает задачу за O(n). Идея: вставляем `#` между символами (`"aba"` → `"#a#b#a#"`), тогда все палиндромы становятся нечётными. Массив `P[i]` хранит радиус палиндрома с центром в `i`. Ключевой трюк: если `i` внутри уже найденного палиндрома, его радиус можно оценить по зеркальному отражению, не раскрывая с нуля. Полная реализация — строк 20, но на собесе просят редко.
