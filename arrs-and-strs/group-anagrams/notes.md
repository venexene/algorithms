# Group Anagrams

## Варианты решения

### Хеш-таблица с сортировкой ключа — O(n · k log k), O(n · k)
Для каждой строки: сортируем символы — получаем ключ, одинаковый для всех анаграмм. Мапа: ключ → слайс строк. В конце собираем все слайсы в результат.

### Хеш-таблица с частотным ключом — O(n · k), O(n · k)
Вместо сортировки — массив частот символов `[26]int`. Go позволяет использовать массив как ключ мапы напрямую — `map[[26]int][]string`. Без конвертации в строку, без разделителей, без коллизий.

## Что важно запомнить
- Не вызывай одну и ту же функцию дважды в одной строке для получения ключа — сохрани в переменную.
- `[26]int` как ключ мапы — идиоматично для Go. Не надо склеивать частоты в строку через `strconv.Itoa`.
- Склейка частот без разделителя даёт коллизии: `[1,11]` и `[11,1]` → одинаковая строка `"111..."`. С массивом как ключом этой проблемы нет.
- `make([][]string, 0, len(mp))` — правильный паттерн для сбора результата без лишних реаллокаций.

## Связь с Valid Anagram
По сути задача — применить Valid Anagram ко всем парам строк, но хеш-таблица по ключу-анаграмме делает это за O(n) группировок вместо O(n²) сравнений. Тот же паттерн «частотный словарь как идентификатор», просто применённый массово.
