# Valid Anagram

## Варианты решения

### 1. Сортировка — O(n log n), O(1) или O(n)
Отсортировать обе строки и сравнить. Если равны — анаграмма.
Минус: O(n log n) на сортировку, плюс нужно аккуратно работать со строками (длина, конвертация в слайс рун/байт).

### 2. Подсчёт частот через массив — O(n), O(1)
Массив на 26 элементов (латиница). Для каждого символа `s` увеличиваем счётчик, для `t` — уменьшаем. В конце все элементы должны быть 0.
Плюсы: один проход, константная память, без мапы. Минус: только для фиксированного алфавита.

### 3. Подсчёт частот через хеш-таблицу — O(n), O(k)
То же самое, но с мапой вместо массива. Универсальнее — работает с Unicode. Но массив быстрее и проще для этой задачи.

## Что важно запомнить
- Сначала проверка длин: `len(s) != len(t)` → сразу `false`. Защита от паники при сравнении.
- Массив фиксированного размера быстрее мапы — прямой доступ по индексу, нет хеширования и аллокаций.
- Один массив на два счётчика (`count[s[i]]++` и `count[t[i]]--`) элегантнее, чем две мапы.
- Сортировка строк в Go: проще через `[]rune` + `sort.Slice`, а не `strings.Split` + `sort.Strings` + `strings.Join`.
