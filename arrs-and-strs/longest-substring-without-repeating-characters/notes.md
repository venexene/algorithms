# Longest Substring Without Repeating Characters

## Варианты решения

### Sliding Window с мапой позиций — O(n), O(min(n, alphabet))
Окно `[left, right]` — текущая подстрока без повторов. Мапа хранит последнюю позицию каждого символа. Когда встречаем символ, который уже в окне — сдвигаем `left` за его предыдущую позицию. На каждом шаге обновляем максимум.

Можно также через мапу частот с посимвольным сужением окна (сдвигать `left` пока частота не станет 1), но мапа позиций элегантнее — один прыжок вместо цикла.

## Как работает окно

```
s = "abcab"
     ↑↑
     LR      окно="a",  max=1
     ↑ ↑
     L R      окно="ab", max=2
     ↑  ↑
     L  R      окно="abc",max=3
       ↑  ↑
       L  R    'a' уже есть на позиции 0 → left=0+1=1, окно="bca", max=3
         ↑  ↑
         L  R  'b' уже есть на позиции 1 → left=1+1=2, окно="cab", max=3
```

## Краевые случаи и как они ломали решение

### 1. `val >= left`, не `val > left`
Символ на позиции `left` УЖЕ в окне. Если `val == left` — это повтор, окно надо сдвигать.
```
s = "aab"
right=1: mp['a']=0, left=0. 0 > 0? Нет → пропустили, окно осталось "aa"
```

### 2. Мапа должна обновляться всегда, не только в else
Когда `left` сдвинулся, старый индекс в мапе остался. При следующей встрече символа сравнение `val >= left` может не сработать корректно.
```
s = "abba"
right=2: 'b' повторился, left сдвинулся, но mp['b'] остался = 1 (старый)
right=3: mp['a']=0, 0 >= left(2)? Нет → считаем что 'a' не в окне
         Но ОК в этом примере работает, однако мапа с мусорными значениями — бомба
```
Решение: `mp[ch] = right` без условий, после проверки на повтор.

### 3. Длина окна: `right - left + 1`, не `right - left`
Разница индексов даёт количество ПРОМЕЖУТКОВ, а не элементов. +1 включает правый край.
```
left=0, right=2 → элементов 3, а не 2
```

## Что важно запомнить
- Sliding window с мапой позиций — основной паттерн для «длиннейшая/кратчайшая подстрока с условием»
- Мапа хранит ПОСЛЕДНЮЮ позицию символа — это позволяет перепрыгнуть через все дубликаты разом
- Не надо двигать `left` по одному — мапа даёт сразу новую позицию
- `>=`, не `>` при сравнении с `left` — граница включена в окно
- Обновляй мапу всегда, не только когда условие ложно

## Родственные задачи
- Minimum Window Substring — sliding window с двумя мапами
- Permutation in String — фиксированное окно + частоты
- Longest Repeating Character Replacement — окно с максимумом частоты
