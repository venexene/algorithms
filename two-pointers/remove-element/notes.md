# Remove Element

## Решение

### Write-pointer — O(n), O(1)
Один указатель `pos` — куда писать следующий «хороший» элемент. Проходим по массиву: если `nums[i] != val` — пишем в `nums[pos]`, сдвигаем `pos`. Возвращаем `pos` — он и есть `k`.

Тот же паттерн что Remove Duplicates from Sorted Array: read/write pointers, in-place, возврат позиции записи.

## Что пошло не так
1. Пытался решить свапами с двух концов — `right` уходил в отрицательные, паника
2. Искал первую позицию `val` отдельным циклом — дублировал обработку, `k` завышался
3. `return pos + 1` вместо `pos` — `pos` и так равен количеству оставленных элементов

## Что важно запомнить
- Write-pointer паттерн: `pos` начинается с 0, инкрементится только когда пишем. После цикла `pos == k`.
- Не усложняй: не надо свапов, не надо поиска первого val, не надо двух указателей с концов. Просто перезапись.
- Задача лёгкая, но после усталости ломается даже лёгкое. Когнитивная усталость реальна.
