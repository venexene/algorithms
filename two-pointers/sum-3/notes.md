# 3Sum

## Варианты решения

### 1. Сортировка + two pointers — O(n²), O(1) / O(k)
Сортируем массив. Внешний цикл по `i`, внутренний — two pointers для `left` и `right` как в Two Sum II. Дедупликация без сета: пропускать одинаковые значения на каждом уровне.

### 2. Хеш-таблица (Two Sum внутри) — O(n²), O(n)
Для каждого `i` запускаем Two Sum через мапу для оставшейся части массива. Работает, но медленнее из-за аллокации мапы на каждом шаге и `slices.Sort` для каждой тройки.

## Что важно запомнить

### Дедупликация без сета
- `i`: `if i > 0 && nums[i] == nums[i-1] → continue` (защита `i > 0` обязательна)
- `left`: после нахождения тройки `for left < right && nums[left] == nums[left-1] { left++ }`
- `right`: аналогично `for left < right && nums[right] == nums[right+1] { right-- }`

### Почему мапа медленнее
- Новая `seen := map[int]int{}` на каждой итерации `i` — n аллокаций
- `slices.Sort` на каждую найденную тройку
- Конвертация `[3]int` → `[]int` в конце
Асимптотически одинаково (O(n²)), но константа выше.

### LeetCode проценты — шум
Разница между 60% и 99% на этой задаче — 10–15 мс. Рантайм Go, jitter сервера, фаза GC влияют сильнее чем алгоритм. Оптимальное решение может показывать 60%.

## Связь с Two Sum и Two Sum II
3Sum = внешний цикл + Two Sum II внутри. Фиксируем первый элемент, для оставшейся части ищем пару с суммой `-nums[i]`. Отличие от Two Sum II: нужна дедупликация, потому что троек может быть несколько.
