# Implement Queue using Stacks

## Решение

### Два стека, перегонка при Push — Push O(n), Pop O(1)
Push: перегоняем main → func, кладём новый элемент, перегоняем func → main. Pop/Peek: просто с main стека.

### Амортизированный вариант — O(1) amortized
Два стека: input и output. Push всегда в input. Pop/Peek: **если** output пуст — перегоняем весь input в output. Элементы не возвращаются обратно. Амортизированное O(1), потому что каждый элемент перегоняется не более одного раза.

## Что важно запомнить
- Перегонка только когда output пуст, и только в одну сторону. Обратная перегонка ломает амортизацию.
- `Empty` проверяет оба стека.
- `Pop` без проверки `output.Empty()` перед перегонкой — элементы из input падают поверх оставшихся в output, ломая порядок очереди.

## Что пошло не так
1. Перегонял при Push — O(n) каждую вставку
2. Перегонял обратно после Pop — амортизация пропадала
3. Перегонял всегда (без проверки `output.Empty()`) — ломался порядок при смешанных Push/Pop
4. `Empty` проверял только один стек
