# Permutation in String

## Решение

### 1. Sliding window + `map[rune]int` + полная проверка — O(n), O(1)
Рабочий первый вариант. Мапа частот s1. Фиксированное окно. На каждом шаге сканируем всю мапу: если все `v == 0` — перестановка найдена. Бьёт ~15%.

### 2. Sliding window + `[26]int` + transitions counter — O(n), O(1)
Оптимизация. Массив вместо мапы — прямой доступ. Счётчик `count` — число позиций массива, равных 0. `need` — число уникальных символов в s1. Когда `count == need` — окно содержит перестановку. Без сканирования на каждом шаге. Бьёт 100%.

## Что важно запомнить
- `need` — количество уникальных символов, не `len(s1)`. `"aa"` → need=1, len=2.
- Счётчик переходов через 0: `count++` когда значение стало 0, `count--` когда перестало быть 0. Не на каждом изменении.
- Массив `[26]int` быстрее мапы: прямой доступ по индексу, нет хеширования.
- `s2[i]-'a'` — байты напрямую, без конвертации в `rune`.

## Что пошло не так
1. `if v == 1 { break } return true` — логика наоборот (ошибка в первом варианте)
2. `if v == 1` впринцпе неверная проверка и нужно было проверять неравенство нулю (ошибка в первом варианте)
3. `count == len(s1)` — ломается на дубликатах в s1
4. `count--` без проверки перехода через 0 — срабатывал всегда при ненулевом значении
5. Итог: 100% после исправления всех трёх
