# Top K Frequent Elements

## Решение

### Hash map + min-heap — O(n log k), O(n + k)
Две фазы:
1. Мапа: подсчёт частот всех чисел
2. Min-heap размера k по частоте: проходим по мапе, кладём числа в кучу. Когда размер > k — удаляем минимальный.

В куче остаются k наиболее частых. Min-heap, потому что удаляем наименее частый — остаются топ.

### Bucket sort — O(n), O(n)
Массив корзин размером `n+1`, где индекс = частота. Раскладываем числа из мапы по корзинам. Проходим от конца к началу, собираем элементы пока `len(res) < k`. Важно: идём пока не набрали k ЭЛЕМЕНТОВ, а не k корзин — пустые корзины не должны считаться.

## container/heap в Go

В отличие от других языков, Go требует чтобы ты реализовал интерфейс `heap.Interface` на СВОЁМ типе:
- `Len()`, `Less()`, `Swap()` — для сравнения и перестановки
- `Push(x any)` — добавляет элемент (ты управляешь слайсом)
- `Pop() any` — удаляет последний (ты управляешь слайсом)

А затем вызываешь ПАКЕТНЫЕ функции `heap.Push(h, x)` и `heap.Pop(h)` — они используют твои методы, но дополнительно делают `up`/`down` для поддержания свойства кучи.

### Частая ошибка
```go
h.heap.Pop()  // метод структуры — удаляет последний, НЕ чинит кучу
heap.Pop(h.heap)  // пакетная функция — свапает, чинит, потом удаляет
```

### Value vs pointer receiver
`Len`, `Less`, `Swap` с value receiver — ок, если внутри ссылочные типы. `Push`, `Pop` — только pointer receiver (меняют слайс).

## Что важно запомнить
- Две фазы разделены: сначала счёт, потом отбор. Не смешивай — куча не умеет считать частоты одновременно с отбором.
- Min-heap для топ-k: удаляешь минимум → остаются максимумы.
- `{0: 1}` в мапе частот НЕ нужен здесь — в отличие от Subarray Sum, нет пустого префикса.

## Что пошло не так
1. Пытался считать частоты И отбирать топ-k внутри кучи одновременно — фазы смешались. Push в кучу из nums, Pop при превышении k, частота в мапе росла но элементы удалялись — каша.
2. Вызывал `h.heap.Pop()` вместо `heap.Pop(h.heap)` — `FreqHeap.Pop()` просто удаляет последний, не чинит кучу. Пакетная `heap.Pop()` сначала свапает, делает down, потом вызывает Pop.
3. Тень переменной `k` — `for k := range mp` перекрывает параметр `k`.
4. Bucket sort: `for i := len-1; i > len-1-k; i--` — отсчитывал КОРЗИНЫ, а не элементы. Пустые корзины считались как итерация, но не добавляли чисел в результат.
